## 1. 도입: 왜 딥러닝 라이브러리를 써야 할까요?

지금까지 우리는 텐서플로우(TensorFlow)라는 거대한 도구 상자 안에서, 케라스(Keras)라는 아주 쓰기 편한 리모컨을 가지고 딥러닝 모델(DenseNet, CNN 등)을 조립해 왔습니다. 

하지만 세상에는 우리가 배운 것 외에도 수많은 딥러닝 도구 상자들이 존재하며 매년 새로운 것들이 쏟아져 나옵니다. 만약 이런 도구 상자가 없다면 어떨까요? 우리는 인공지능이 계산할 수백만 개의 행렬 미적분 공식을 C언어나 어셈블리어로 밑바닥부터 0과 1을 치며 하드코딩해야 할 것입니다. 

다행히도 구글, 메타(페이스북) 같은 거대 기업들이 이 복잡하고 고통스러운 미적분 수학과 그래픽 카드(GPU) 통신 코드를 하나의 패키지로 깔끔하게 묶어서 공짜로 배포했습니다. 이것이 바로 **'딥러닝 라이브러리(Deep Learning Library)' 혹은 '프레임워크(Framework)'**입니다. 

이번 시간에는 현재 전 세계 AI 업계를 반분하고 있는 두 거인, **TensorFlow**와 **PyTorch**의 특징과 역사, 그리고 실무에서 어떤 것을 선택해야 하는지 아주 깊고 자세하게 해부해 보겠습니다.

## 2. 절대강자 TensorFlow (구글) - "운영 환경(Production)의 지배자"

텐서플로우(TensorFlow)는 아마도 대중에게 가장 널리 알려진 딥러닝 라이브러리일 것입니다. 2015년에 구글이 자신들의 내부 프로젝트를 위해 개발하다가 세상에 오픈소스로 공개했습니다.

### 💡 TensorFlow의 이름 뜻은 무엇일까요?
*   **Tensor (텐서)**: 우리가 딥러닝에 집어넣는 데이터(이미지 픽셀, 숫자의 배열)를 부르는 수학적 용어입니다.
*   **Flow (플로우)**: 이 데이터(Tensor)들이 인경신경망의 뉴런들(`w*x + b`)을 마치 컨베이어 벨트를 타듯 쭈르륵 흘러가며(Flow) 계산된다는 뜻입니다.

### 📜 TensorFlow 1.x의 흑역사와 2.0의 혁명
사실 옛날 옛적 텐서플로우 1.x 버전은 배우기가 **끔찍하게 고통스러웠습니다.** 
당시에는 모델을 만들 때 `3단계 절차`를 강제로 거쳐야 했습니다.
1.  **그래프 그리기**: "아, 데이터가 이렇게 들어오면 여기서 곱하고 저기서 더하자"라는 수학적 회로도(Computational Graph)를 종이에 그리는 과정.
2.  **세션(Session) 열기**: 컴퓨터와 통신을 시작하는 징을 울림.
3.  **데이터 들이붓기**: 그제야 데이터를 쏟아붓고 결과를 확인.

왜 이렇게 복잡하게 만들었을까요? 구글은 AI를 수만 대의 컴퓨터(CPU, GPU, 클라우드 서버)에 동시에 쪼개서 실행(분산 처리)해야 했는데, 이렇게 회로도를 미리 고정시켜놓고 실행해야만 컴퓨터 렉이 덜 걸리고 속도가 미친 듯이 빨랐기 때문입니다. (최적화의 극치)

하지만 경쟁자인 PyTorch가 너무나 직관적이고 편한 방식으로 치고 올라오자, 위기를 느낀 구글은 **TensorFlow 2.0 대규모 업데이트**를 단행합니다.
*   고통스러운 3단계를 다 없애버리고, 우리가 앞선 단원들에서 너무나 편하게 썼던 **`Keras(케라스)`를 공식 메인 리모컨으로 흡수 통합**시켜 버렸습니다.
*   **즉시 실행(Eager Execution) 모드**를 기본으로 탑재하여, 파이썬 코드 한 줄을 칠 때마다 즉각적으로 결괏값을 볼 수 있게 만들었습니다.

### 🏭 TensorFlow가 실무(비즈니스)에서 절대적인 이유
텐서플로우의 가장 무서운 점은 단지 '모델을 훈련'시키는 것을 넘어, 다 훈련된 AI를 실제 소비자들의 휴대폰이나 웹사이트에 서비스로 올릴 때 필요한 **생태계(Ecosystem)**가 미친 듯이 잘 구축되어 있다는 점입니다.
*   **TensorFlow Serving**: 만든 AI 모델을 회사 클라우드 서버에 안정적으로 올려서 수백만 명의 접속을 동시에 처리할 때 씁니다.
*   **TensorFlow Lite**: 만든 AI를 용량을 극한으로 깎고 최적화해서, 인터넷이 안 터지는 테슬라 자율주행차 내부 칩셋이나, 여러분의 스마트폰 앱 속에 가볍게 우겨넣을 때 씁니다.
*   **TensorFlow.js**: AI를 웹브라우저(크롬, 사파리) 안에서 돌아가게 만듭니다. 사용자는 따로 프로그램을 깔 필요 없이 웹사이트에 접속하는 것만으로 내 AI를 쓸 수 있습니다.

## 3. 떠오르는 맹수 PyTorch (메타) - "연구자(Research)들의 최애 픽"

텐서플로우가 구글의 작품이라면, 파이토치(PyTorch)는 페이스북(현 메타)의 AI 연구팀(얀 르쿤 교수 주도)이 만든 작품입니다. 'Torch'라는 오래된 C언어 기반 라이브러리를 **파이썬(Python) 개발자들이 쓰기 가장 편하도록 완전히 새롭게 뜯어고친(Pythonic) 걸작**입니다.

### 🐍 파이썬과 혼연일체 (NumPy와의 호환성)
파이토치의 가장 큰 특징은 우리에게 친숙한 파이썬의 숫자 계산 라이브러리인 **NumPy(넘파이)와 너무나도 똑같이 생겼다**는 점입니다. 
파이토치의 텐서(데이터 덩어리)는 넘파이 배열과 완벽하게 호환되며 자유롭게 변환할 수 있습니다. 그래서 파이썬을 다룰 줄 아는 실무자라면 아무런 거부감 없이 파이토치 코드를 짤 수 있습니다. 단지 차이점이 있다면 넘파이는 CPU에서만 돌지만, **파이토치는 이 계산을 그래픽카드(GPU)에 얹어서 수천 배 빠르게 병렬 연산**을 해준다는 것입니다.

### 🛠 동적 계산 그래프 (Dynamic Computational Graph)와 미친 디버깅 속도
텐서플로우(1.x)가 한 번 건물을 지으면 바꿀 수 없는 '정적(Static)'인 모델이었다면, 파이토치는 코드가 실행되는 그 순간순간 유연하게 건물을 짓고 부술 수 있는 **'동적(Dynamic)' 모델**입니다.
*   에러가 났을 때 텐서플로우는 에러 메시지가 너무 복잡해서 머리가 터질 것 같았습니다.
*   파이토치는 파이썬의 표준 디버깅 툴을 그대로 쓸 수 있어서, 에러가 어느 줄에서 왜 났는지(Stacktrace) 즉각적으로 확인하고 고칠 수 있습니다. 그래서 새로운 아이디어를 실험하고 뚝딱뚝딱 수정해야 하는 **학계 논문 연구자들의 70% 이상이 현재 파이토치를 사용**합니다.

## 4. TensorFlow vs PyTorch 한눈에 비교하기

자, 그럼 실무자로서 어떤 것을 배워야 할까요? 표를 통해 한눈에 정리해 드립니다.

| 비교 항목 | TensorFlow (Google) | PyTorch (Meta) |
| :--- | :--- | :--- |
| **탄생 배경** | C++ 기반의 엔진을 파이썬으로 쓸 수 있게 포팅함. | NumPy와 유사하게 파이썬 코더들이 직관적으로 쓰도록 완전히 새로 짬. |
| **그래프 방식** | 정적 그래프(Static) 위주. (물론 2.0부터는 Eager 모드로 동적이 됨) | 동적 미분(Dynamic Auto-differentiation) 기본 탑재. 짱 유연함. |
| **디버깅(에러 잡기)** | 과거에는 지옥 같았고 지금도 꽤 복잡함. | 파이썬 기본 기능이라 아주 직관적이고 쉬움. |
| **고수준 API** | `Keras` (가장 완벽한 밸런스의 리모컨) | `fast.ai` (너무 쉬워서 오히려 세밀한 조작이 힘들 수 있음) |
| **최고의 장점** | 만든 모델을 실제 고객의 폰이나 웹으로 상용화(배포)하기 너무 좋음. | 새로운 논문을 구현하거나, 기발한 형태의 실험을 하기 너무 좋음. |

*   **전문가의 결론**: **"둘 다 배워라!"** 
    실무에 나가보면 리서치 팀은 PyTorch로 실험을 하고, 서비스 배포 팀은 그것을 TensorFlow로 변환해서 앱에 올리는 경우가 허다합니다. 둘 다 철학이 다를 뿐 매우 비슷하기 때문에, 하나를 깊게 이해하면 나머지는 문법만 1주일 공부하면 바로 쓸 수 있습니다. (하지만 굳이 하나를 꼽자면 **연구/실험 목적이면 PyTorch**, **상용화/취업 목적이면 TensorFlow**를 추천합니다.)

## 5. 그 외의 단역들 (알아만 두세요!)

빅테크 기업들의 전쟁에서 밀려났지만 한때 이름을 날렸던 도구들도 잠시 소개합니다.
*   **MXNet (아마존)**: 아마존 AWS 클라우드에서 밀어주는 도구입니다.
*   **CNTK (마이크로소프트)**: MS가 만든 도구로, 음성 인식 쪽에서 잠시 반짝했습니다.
*   **Caffe (UC 버클리 / 페이스북)**: 오직 컴퓨터 비전(CNN 이미지 처리)만을 위해 깎아 만든 극단적인 장인정신의 도구입니다. 이후 Caffe2로 페이스북에 흡수되었다가 결국 PyTorch 프로젝트에 통합되었습니다.
*   **Theano (몬트리올 대학)**: 텐서플로우가 나오기 전 세상을 지배하던 할아버지 격 라이브러리입니다. Keras가 원래 이 Theano를 편하게 쓰려고 만들어진 리모컨이었습니다. 지금은 개발진이 모두 구글로 이직하면서 개발이 중단(사망)되었습니다.

## 6. [실전 딥다이브] 코딩 배틀: LeNet-5 (TensorFlow vs PyTorch)

말씀드린 대로, 다음 단원부터는 PyTorch를 배우게 될 것입니다. "대체 파이토치는 코드를 어떻게 치길래 다들 열광하는 걸까?" 
여러분이 이전 4단원에서 열심히 짰던 Keras 기반의 **실전형 LeNet-5 (CNN 이미지 분류기)**를, 이번에는 Keras와 PyTorch 두 가지 버전으로 동시에 나란히 비교해 보겠습니다. 코드를 훑어보기만 해도 두 라이브러리의 냄새(스타일) 차이를 확실히 느끼실 수 있을 것입니다.

### 6.1. TensorFlow (Keras) 버전 - "레고 블록처럼 쌓기"
우리가 너무나 잘 아는 `Sequential` 방식입니다. 순서대로 레이어를 착착착 넣기만 하면 알아서 조립됩니다.

```python
import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Conv2D, MaxPooling2D, Flatten, Dense

# Keras는 모델 뼈대를 하나 선언하고 밑으로 계속 add()를 하며 레고를 쌓습니다.
model = Sequential()

# 첫 번째 합성곱 레이어
model.add(Conv2D(32, kernel_size=(3, 3), activation='relu', input_shape=(28, 28, 1)))
# 풀링 레이어
model.add(MaxPooling2D(pool_size=(2, 2)))

# 평탄화 및 분류기 레이어
model.add(Flatten())
model.add(Dense(128, activation='relu'))
model.add(Dense(10, activation='softmax'))

# Keras의 최고 장점! 컴파일과 핏(Fit) 함수 하나로 모든 훈련이 딸깍! 끝납니다.
model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
# model.fit(X_train, y_train, epochs=10)
```

### 6.2. PyTorch 버전 - "객체지향 설계와 직접 통제하기"
파이토치는 레고를 그냥 쌓는 게 아니라, **파이썬의 클래스(Class)**를 선언해서 뼈대를 직접 만듭니다. `__init__`에서 부품을 사 오고, `forward`에서 부품들을 어떻게 조립해서 데이터를 흘려보낼지 수동으로 배관 공사를 해주는 느낌입니다.

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# PyTorch는 반드시 Python의 Class 형태로 모델을 정의합니다.
class LeNet5_PyTorch(nn.Module):
    def __init__(self):
        super(LeNet5_PyTorch, self).__init__()
        # 1. 부품 사오기 (__init__ 단계)
        # 1입력 채널(흑백), 32출력 필터, 3x3 사이즈
        self.conv1 = nn.Conv2d(in_channels=1, out_channels=32, kernel_size=3)
        # 풀링 부품
        self.pool = nn.MaxPool2d(kernel_size=2, stride=2)
        # Dense 부품 (Linear라고 부름). 
        # Flatten 후 크기를 직접 수학적으로 계산해서 적어줘야 합니다! (여기가 Keras보다 까다로움)
        self.fc1 = nn.Linear(in_features=32 * 13 * 13, out_features=128)
        self.fc2 = nn.Linear(in_features=128, out_features=10)

    def forward(self, x):
        # 2. 부품 조립하기 (데이터가 흐르는 길을 직접 코딩)
        x = self.conv1(x)         # 합성곱 통과
        x = F.relu(x)             # ReLU 마법 통과
        x = self.pool(x)          # 풀링 통과
        
        # Flatten 과정: PyTorch에서는 view나 flatten 함수로 수동으로 1차원으로 찢어줍니다.
        x = torch.flatten(x, 1)   
        
        x = self.fc1(x)           # 첫 번째 Dense(Linear) 통과
        x = F.relu(x)
        x = self.fc2(x)           # 최종 출력
        
        # 참고: PyTorch는 손실 함수(CrossEntropyLoss) 안에 Softmax가 내장되어 있어서 
        # 모델의 맨 끝에 Softmax를 따로 적지 않는 것이 실무 관행입니다.
        return x

# 모델 객체 생성
model_pytorch = LeNet5_PyTorch()
print(model_pytorch)

# Keras의 model.fit() 같은 편리한 마법 버튼이 없습니다!
# 연구자가 for문을 돌면서 "순전파 -> 오차 계산 -> 역전파 기울기 계산 -> 가중치 업데이트" 
# 라는 수학적 흐름을 4~5줄의 코드로 매번 직접 짜줘야 합니다. (다음 5.2강에서 배우게 됩니다!)
```

### 💡 실무 코딩 스타일 요약
*   **Keras (TensorFlow)**: `Sequential`로 상자 안에 때려 넣고, `.compile()`과 `.fit()`이라는 마법 버튼 2개만 누르면 커피를 마시고 와도 될 정도로 자동화되어 있습니다. 쉽고 빠릅니다.
