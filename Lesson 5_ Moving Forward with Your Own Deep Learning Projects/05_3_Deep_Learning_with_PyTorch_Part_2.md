## 1. 수동 훈련 루프(Training Loop) 심층 해부

이전 파트에서 우리는 파이토치 훈련의 핵심인 **'심장 박동 5단계'** 코드를 미리 맛보았습니다. Keras에서는 `.fit()` 버튼 하나로 숨겨져 있던 이 위대한 수학적 흐름을, 이제 한 줄 한 줄 뜯어보며 완벽하게 내 것으로 만들어 봅시다.

### 🏃‍♂️ 순전파 (Forward Propagation)
데이터가 모델을 뚫고 지나가서 예측값을 만들어내는 과정입니다.
```python
# 1. 28x28 이미지를 784개의 1차원 선으로 쫙 폅니다. (-1 꼼수 기억하시죠?)
x_flat = images.view(images.shape[0], -1)

# 2. 펴진 데이터를 우리가 만든 모델에 집어넣어 예측값(y_hat)을 뽑아냅니다.
y_hat = model(x_flat)

# 3. 채점관(CrossEntropyLoss)을 불러와서, 내 예측(y_hat)과 실제 정답(labels)의 오차(Cost)를 계산합니다.
loss = criterion(y_hat, labels)
```

### 🔙 역전파 (Backward Propagation) & 가중치 업데이트
오차를 알았으니, 이제 모델의 뇌 세포(가중치와 편향)를 고쳐서 더 똑똑하게 만들 차례입니다.
```python
# 4. [매우 중요] 최적화 요정의 기억 지우기
# 파이토치는 특이하게도 이전 배치의 기울기(미분값)를 계속 누적(Accumulate)하는 성질이 있습니다.
# 그래서 새로운 배치를 학습하기 전에는 반드시 쓰레기통을 비워줘야 합니다!
optimizer.zero_grad()

# 5. 역전파 마법 실행
# 이 한 줄이 딥러닝의 꽃입니다. 오차(loss)를 뒤로 흘려보내서(backward), 
# 모델 내부의 수만 개의 가중치들이 '내가 얼마나 잘못했는지(기울기)'를 각각 계산하게 만듭니다.
loss.backward()

# 6. 가중치 업데이트
# 최적화 요정(SGD 등)이 나서서, 방금 계산된 기울기 방향으로 가중치들을 살짝(학습률만큼) 이동시킵니다.
optimizer.step()
```
이 과정을 미니배치(128장)마다 계속 반복하면, 모델은 점차 정답을 향해 진화하게 됩니다.

## 2. 실무 팁: 검증 루프 (Validation Loop) 만들기

훈련이 끝났으면, 이 모델이 처음 보는 데이터(Test Data)에서도 잘 작동하는지 시험을 쳐야겠죠? 
Keras에서는 `.evaluate()`라는 버튼 하나면 끝났지만, 파이토치에서는 시험을 치는 환경(Validation Loop)도 우리가 직접 세팅해 주어야 합니다.

이때 **가장 많이 실수하는 2가지 마법의 주문**이 있습니다. 무조건 외우셔야 합니다!

```python
# 마법의 주문 1: "이제부터 실전(평가) 모드다!"
model.eval() 
```
*   **왜 `model.eval()`을 쓰나요?** 우리가 신경망을 훈련할 때는 과적합을 막기 위해 뇌세포 일부를 기절시키는 `Dropout(드롭아웃)`이나, 배치의 평균을 맞추는 `BatchNorm(배치 정규화)`을 사용합니다. 하지만 **시험을 칠 때는 기절했던 뇌세포를 100% 다 깨워서 온 힘을 다해 문제를 풀어야 합니다.** `model.eval()`은 "이제 시험 칠 거니까 드롭아웃 기능 꺼!"라고 모델에게 지시하는 스위치입니다. (훈련을 다시 시작할 때는 `model.train()` 스위치를 켜주면 됩니다.)

```python
# 마법의 주문 2: "이제부터 기울기(미분) 계산하지 마!"
with torch.no_grad():
    for images, labels in test_loader:
        # 시험 치는 코드...
```
*   **왜 `torch.no_grad()`를 쓰나요?** 훈련할 때는 가중치를 고치기 위해 미분(Gradient)을 엄청나게 계산하고 메모리에 저장해 둡니다(Autograd 기능). 하지만 **시험을 칠 때는 가중치를 고칠 필요가 없이 그냥 정답만 맞추면 됩니다.** 이 주문을 걸면 파이토치가 미분 계산 엔진을 끄기 때문에, 메모리(RAM) 소비가 확 줄어들고 속도가 미친 듯이 빨라집니다.

## 3. 깊은 신경망(Deep Net)으로의 진화 (실전 아키텍처)

얕은 신경망은 끝났습니다. 이제 실무에서 폼나게 쓸 수 있는 은닉층 3개짜리 '깊은 신경망(Deep Neural Network)'을 파이토치로 조립해 봅시다. Keras에서 만들었던 구조와 똑같습니다.

```python
class DeepNet_PyTorch(nn.Module):
    def __init__(self):
        super(DeepNet_PyTorch, self).__init__()
        
        # 1번째 은닉층: 784 -> 64
        self.fc1 = nn.Linear(784, 64)
        
        # 2번째 은닉층: 64 -> 64
        self.fc2 = nn.Linear(64, 64)
        
        # 3번째 은닉층: 64 -> 64
        self.fc3 = nn.Linear(64, 64)
        
        # 과적합 방지용 Dropout 부품 구매 (50% 확률로 뉴런 기절)
        self.dropout = nn.Dropout(0.5)
        
        # 마지막 출력층: 64 -> 10 (숫자 0~9)
        self.out = nn.Linear(64, 10)

    def forward(self, x):
        x = x.view(x.shape[0], -1) # 일렬로 쫙 펴기
        
        # 1층 통과 (실무의 표준인 ReLU 활성화 함수 사용!)
        x = F.relu(self.fc1(x))
        
        # 2층 통과
        x = F.relu(self.fc2(x))
        
        # 3층 통과 후, 가혹한 시련(Dropout) 부여
        x = F.relu(self.fc3(x))
        x = self.dropout(x)
        
        # 최종 출력
        x = self.out(x)
        return x

model = DeepNet_PyTorch()
# 이번엔 단순한 SGD 대신, 실무 최고의 똑똑한 요정인 'Adam'을 고용합니다!
optimizer = optim.Adam(model.parameters(), lr=0.001)
```
파이토치의 진정한 매력이 느껴지시나요? `__init__`에서 부품을 마음대로 사 오고, `forward`에서 레고 조립하듯 데이터를 요리조리 통과시키는 이 **'완벽한 통제감'**이 바로 전 세계 연구자들이 파이토치를 사랑하는 이유입니다.

## 4. 🕵️‍♂️ 딥러닝 최대의 미스터리: "왜 테스트 점수가 훈련 점수보다 높게 나오죠?"

파이토치 코드를 돌려보거나 딥러닝을 처음 접하시는 분들이 가장 많이 하는 질문이 있습니다.
> *"과적합(Overfitting)이 되면 훈련 점수가 더 높고 테스트 점수가 낮아진다는 건 배웠어요. 근데 반대로 **훈련 정확도(Training Accuracy)는 92%인데, 테스트 정확도(Validation Accuracy)는 95%**로 더 높게 나옵니다! 제 코드가 고장 난 건가요?"*

고장 난 것이 아닙니다! 이는 아주 자연스러운 현상이며, 그 이유는 **'점수를 측정하는 타이밍'** 때문입니다. 아래 그래프를 보시죠.

![훈련 정확도와 검증 정확도의 미스터리 해답](/Users/shinwookkang/.gemini/antigravity/brain/cb5306ae-e4e0-4381-a3ab-20a49944023c/val_acc_vs_train_acc_1784005073645.jpg)

### 💡 미스터리 해답: "평균의 함정"
1.  **훈련 정확도 (Training Accuracy) 계산법**: 
    1 에폭(Epoch)은 미니배치를 469번 반복하는 과정입니다. 훈련 정확도는 1번 배치를 풀었을 때의 점수부터 469번 배치를 풀었을 때의 점수를 **모두 합쳐서 '평균'을 낸 값**입니다. 에폭 초반(1번 배치)에는 모델이 바보 상태였기 때문에 점수가 엉망진창입니다. 즉, **초반의 바보 같았던 점수들이 평균을 다 깎아먹기 때문**에 훈련 정확도가 낮게 나옵니다. (위 그래프의 노란색 면적)
    
2.  **검증 정확도 (Validation Accuracy) 계산법**:
    반면 검증(테스트) 평가는 언제 하나요? 469번의 배치를 모두 돌며 모델이 **가장 똑똑하게 성장한 에폭의 맨 마지막 순간(단 한 번)**에 시험을 칩니다. 당연히 1 에폭 내에서 가장 똑똑해진 '완성형' 상태로 시험을 치니 점수가 높을 수밖에 없습니다! (위 그래프의 빨간 별표)

3.  **추가적인 이유 (Dropout)**: 
    훈련할 때는 우리가 `Dropout`으로 뇌세포의 절반을 강제로 기절시키면서 가혹하게 훈련을 시킵니다(모래주머니를 차고 달리기). 반면 테스트할 때는 `model.eval()`을 통해 모든 뇌세포를 다 깨워서 100%의 전력으로 시험을 칩니다(모래주머니 벗고 달리기). 당연히 실전 테스트 때의 퍼포먼스가 더 잘 나올 수밖에 없습니다.

이제 이 미스터리가 완벽히 풀리셨나요?
