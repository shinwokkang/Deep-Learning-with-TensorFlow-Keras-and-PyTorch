## 1. 도입: 전설의 시작 'LeNet-5'와 현대적 재해석 (Modern Twists)

딥러닝의 역사를 공부하다 보면 반드시 마주치게 되는 전설적인 이름이 있습니다. 바로 **LeNet-5 (르넷 파이브)** 입니다. 1998년, 얀 르쿤(Yann LeCun) 교수와 그의 동료들이 우편물에 적힌 손글씨 우편번호를 자동으로 인식하기 위해 개발한 초기 합성곱 신경망(CNN)입니다. 이번 시간에는 텐서플로우(TensorFlow)와 케라스(Keras)를 사용하여 이 역사적인 LeNet-5 구조를 직접 구현해 볼 것입니다. 

하지만 1998년의 코드를 그대로 치지는 않을 것입니다. 오늘날의 컴퓨터는 1998년에 비해 **비교도 할 수 없을 만큼 빠르고 저렴해졌기 때문(Cheap Compute)**입니다. 따라서 우리는 위대한 조상의 뼈대에 **현대적인 무기(Modern Twists)** 들을 장착하여 훨씬 강력한 모델을 만들 것입니다.

### 💡 LeNet-5에 적용할 3가지 현대적 업그레이드
1.  **필터(돋보기)의 개수 대폭 증가**: 원본 LeNet-5는 컴퓨팅 자원이 부족해서 첫 번째 계층에 6개, 두 번째 계층에 16개의 필터밖에 쓰지 못했습니다. 우리는 자원이 넘쳐나므로 **첫 번째 계층에 32개, 두 번째 계층에 64개의 필터**를 화끈하게 투입할 것입니다. (더 많은 필터 = 더 다양한 특징을 섬세하게 잡아냄)
2.  **풀링(Pooling)의 최소화**: 원본은 차원을 줄이기 위해 풀링을 두 번이나 때렸습니다. 하지만 너무 잦은 풀링은 유용한 정보마저 버리게 만들 수 있습니다. 우리는 강력한 컴퓨팅 파워를 믿고 **풀링을 단 한 번만** 사용하여 데이터의 디테일을 더 살릴 것입니다.
3.  **최신 기술의 도입 (ReLU & Dropout)**: 1998년에는 `ReLU` 활성화 함수나 과적합을 막아주는 `Dropout`이라는 개념 자체가 이 세상에 발명되지 않았습니다. 우리는 이 현대 딥러닝의 필수품들을 적극적으로 도입할 것입니다.

## 2. 데이터 전처리의 대혁신: "제발 사진을 1차원으로 찢지 마세요!"

이전 단원에서 DenseNet(밀집 신경망)을 배울 때, 우리는 28x28 픽셀의 이미지 사진을 네트워크에 넣기 위해 **가늘고 긴 784칸(28*28)짜리 1열 기차로 쫙 찢어버렸습니다 (Flatten).** 
*   **문제점**: 이렇게 하면 픽셀들의 상하좌우 '공간적인 위치 정보'가 완전히 산산조각 납니다. "코 바로 밑에 입이 있다"는 중요한 시각적 힌트가 날아가는 것이죠.

하지만 이제 우리는 시각 정보의 스페셜리스트인 **CNN(합성곱 신경망)**을 사용합니다. CNN은 사진의 형태를 있는 그대로 받아들이는 능력이 있습니다.

```python
# [이전 방식 - DenseNet]
# 28x28 사진을 784개의 1차원 배열로 평탄화 (공간 정보 파괴)
X_train = X_train.reshape(60000, 784) 

# [현대 방식 - CNN]
# 28x28의 공간 형태를 그대로 유지! 뒤에 붙은 '1'은 흑백 채널을 의미함.
X_train = X_train.reshape(60000, 28, 28, 1) 
```

**💡 아주 쉬운 차원(Dimension) 설명**
*   **`28, 28`**: 가로 28픽셀, 세로 28픽셀을 의미하는 2차원 평면입니다.
*   **`1` (채널)**: 사진의 두께입니다. 우리가 쓸 데이터(MNIST 손글씨)는 흑백 사진이므로 색상 두께가 1장(명암만 존재)입니다. 만약 강아지 컬러 사진을 쓴다면 빛의 3원색인 빨/초/파(RGB) 정보가 필요하므로 이 자리가 `3`이 됩니다. 즉 `(28, 28, 3)`의 입체적인 직육면체 블록 형태가 되는 것입니다.

## 3. 새로운 마법의 주문들: Keras 핵심 레이어 파헤치기

코드를 짜기 전, 우리가 소환할 3가지 강력한 레이어들을 초보자의 눈높이에서 완벽히 이해해 봅시다.

### ① Conv2D (2차원 합성곱 레이어)
이름 뒤에 `2D`가 붙은 이유는, 우리가 다루는 이미지 사진이 **가로/세로를 가진 2차원 공간**이기 때문에 이 공간 위를 상하좌우로 훑고 지나가기 때문입니다.

*   **잠깐, 컬러 사진은 두께(RGB=3)가 있으니 3D 아닐까요?**
    *   아닙니다! 컬러 사진도 `Conv2D`를 씁니다. 왜냐하면 돋보기(필터)가 이미지를 훑을 때, 앞뒤 공간으로 이동하는 게 아니라 오직 **종이 위를 상하좌우(2차원)로만 움직이기 때문**입니다. 두께가 3장이면 돋보기의 렌즈 알맹이도 3겹으로 만들어서 한방에 찍어 누르며 이동합니다.
*   **그럼 Conv1D와 Conv3D는 언제 쓰나요?**
    *   **Conv1D (1차원)**: 시간이 흐름에 따라 한 줄로 흘러가는 데이터(예: 주식 차트, 사람의 음성 파형, 번역할 문장의 단어 순서)에서 패턴을 찾을 때 씁니다.
    *   **Conv3D (3차원)**: 입체적인 공간을 상하좌우 뿐만 아니라 앞뒤로도 뚫고 들어가며 분석해야 할 때 씁니다. (예: 병원의 뇌 MRI 입체 스캔 사진, 시간이 결합된 동영상 데이터)

*   **파라미터 세팅**
    *   `kernel_size=(3, 3)`: 돋보기의 렌즈 크기를 가로 3, 세로 3픽셀로 정합니다. 한 번에 9개의 픽셀을 묶어서 관찰합니다.
    *   `strides=(1, 1)`: 돋보기를 이동시킬 때, 한 칸씩 촘촘하게 이동합니다. (Keras 기본값)
    *   `padding='valid'`: 이미지 테두리에 0을 덧대지 않습니다. 이렇게 하면 돋보기가 튀어나가지 않게 움직여야 하므로 사진 크기가 조금씩 작아집니다. (Keras 기본값)

### ② MaxPooling2D (맥스 풀링)
어려운 정보는 놔두고, 해상도를 확 줄여서 컴퓨터의 짐을 덜어주는 레이어입니다.
*   `pool_size=(2, 2)`: 2x2(총 4픽셀) 구역에서 가장 강렬한 특징(숫자가 가장 큰 픽셀) 딱 1개만 남기고 나머지 3개는 버립니다. 결과적으로 사진의 크기(해상도)가 가로세로 딱 절반(1/4 면적)으로 줄어듭니다.

### ③ Flatten (평탄화)
특징 추출부(Conv+Pool)의 작업이 모두 끝나면, 결과물은 아주 뚱뚱하고 작은 직육면체 텐서 블록이 됩니다. 하지만 우리의 최종 분류 역할을 맡을 `Dense` 레이어는 **한 줄로 서 있는 1차원 데이터**만 받을 수 있는 외골수입니다.
*   `Flatten()`은 공간 정보를 박살내도 상관없는 이 시점에서, 입체 블록을 **가늘고 긴 1열 기차**로 인정사정없이 쫙 펴주는 브릿지(다리) 역할을 합니다.

## 4. [실전 코드] 최신 실무형 LeNet-5 완벽 구현

자, 이제 배운 이론을 모두 조립하여 실제 현업에서 사용할 법한 깔끔하고 강력한 코드로 짜봅시다.

```python
import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Conv2D, MaxPooling2D, Flatten, Dense, Dropout

# ---------------------------------------------------------
# 1단계: 특징 추출부 (Feature Extractor) - 사진의 패턴을 분석하라!
# ---------------------------------------------------------
model = Sequential()

# 첫 번째 합성곱 계층 (Conv1)
model.add(Conv2D(
    filters=32,               # 32개의 돋보기(필터)를 투입합니다. (원본 LeNet은 6개였음)
    kernel_size=(3, 3),       # 돋보기 크기는 3x3 픽셀입니다.
    activation='relu',        # 현대적 마법인 ReLU를 씌워줍니다.
    input_shape=(28, 28, 1),  # 첫 레이어에는 반드시 입력 사진의 형태(가로, 세로, 흑백1채널)를 알려줘야 합니다.
    padding='valid'           # 여백을 덧대지 않습니다. (출력 사진은 26x26으로 조금 작아집니다)
))

# 두 번째 합성곱 계층 (Conv2)
model.add(Conv2D(
    filters=64,               # 더 복잡한 모양을 찾기 위해 돋보기를 64개로 확 늘립니다.
    kernel_size=(3, 3),       
    activation='relu'
    # 여기서부터는 알아서 모양을 계산하므로 input_shape가 필요 없습니다.
))

# 풀링 계층 (Pooling)
model.add(MaxPooling2D(
    pool_size=(2, 2)          # 해상도를 반토막 냅니다. 정보는 유지하면서 계산량을 확 줄여줍니다!
))

# 드롭아웃 (Dropout) - 1차 방어막
model.add(Dropout(0.25))      # 특징을 너무 맹신하지 않게(과적합 방지), 데이터의 25%를 랜덤하게 기절시킵니다.


# ---------------------------------------------------------
# 2단계: 차원 축소 및 분류기 (Classifier) - 이게 고양이야 강아지야?
# ---------------------------------------------------------

# 평탄화 (Flatten)
model.add(Flatten())          # 뚱뚱해진 3차원 입체 블록을 Dense가 먹기 좋게 1차원 배열로 쫙 찢어줍니다.

# 깊은 은닉층 (Dense)
model.add(Dense(
    units=128,                # 128개의 뉴런이 1차원으로 들어온 특징 점수들을 이리저리 조합하고 회의합니다.
    activation='relu'
))

# 드롭아웃 (Dropout) - 2차 방어막 (더 강하게!)
model.add(Dropout(0.5))       # 깊은 층일수록 쓸데없는 걸 외울 확률이 높으므로 화끈하게 50%를 기절시킵니다.

# 최종 출력층 (Output)
model.add(Dense(
    units=10,                 # 정답 카테고리가 10개(0~9 숫자)이므로 바구니도 딱 10개를 만듭니다.
    activation='softmax'      # 10개 바구니의 점수를 확률(100% 비율)로 예쁘게 변환해 줍니다.
))
```

## 5. 파라미터(Parameter)의 충격적인 진실과 GPU의 기적

`model.summary()`를 찍어보면 매우 충격적이고 흥미로운 사실을 발견할 수 있습니다.

### 😲 "계산은 내가 다 했는데 짐은 네가 다 지네?" (가중치 효율성)
*   우리가 만든 이 모델의 전체 파라미터(학습해야 할 가중치) 개수는 약 **120만 개 (1.2 Million)**입니다.
*   그런데 이 중 **98.3%**의 파라미터가 `Flatten` 직후에 등장하는 첫 번째 `Dense` 레이어에 몽땅 몰려 있습니다!
*   **Conv2D 레이어들의 반전**: 합성곱 계층은 이미지를 상하좌우로 훑으며 엄청나게 많은 수학적 곱셈/덧셈(연산)을 수행합니다. 연산량은 최고로 많지만, 정작 파라미터(가중치)의 개수는 놀랍도록 적습니다. 왜냐하면 하나의 필터(가중치 뭉치)가 이미지 전체를 돌아다니며 **재사용(가중치 공유, Weight Sharing)** 되기 때문입니다. 이는 CNN이 일반 Dense 네트워크보다 훨씬 효율적이고 똑똑한 아키텍처라는 증거입니다.

### 🚀 GPU (그래픽 카드)가 절대적으로 필요한 이유
강의를 보면, 이 모델을 강력한 딥러닝 서버의 CPU로 돌려도 에포크(Epoch)당 무려 25~26초가 걸립니다. DenseNet을 돌릴 땐 1초밖에 안 걸렸는데 말이죠! 
*   **왜 그럴까요?**: Conv2D는 화면 구석구석을 수십 개의 필터로 쓸고 다니며 수백만 번의 자잘한 행렬 곱셈을 해야 합니다. CPU는 똑똑하지만 한 번에 한 가지 일밖에 못하는 엘리트 학자입니다. 
*   **GPU의 위력**: 구글 Colab에서 제공하는 GPU 하드웨어 가속기로 실행하면 이 시간이 단 **3~4초**로 줄어듭니다! GPU는 수만 개의 멍청한 단순 노동자들을 모아놓은 군대와 같아서, 화면 전체 픽셀의 단순 곱셈을 동시에 1초 만에 박살 내버리기 때문입니다. **"이미지 딥러닝을 할 땐 무조건 GPU를 켜라!"**가 이 바닥의 철칙입니다.

### 🏆 놀라운 최종 성적표
기존의 DenseNet은 아무리 깊게 파도 검증 정확도(Validation Accuracy)가 97.6% 정도에서 벽에 부딪혔습니다. 하지만 시각 정보의 제왕인 CNN(우리의 현대판 LeNet)을 돌린 결과, 최고 에포크에서 무려 **99.25%**의 정확도를 달성했습니다! 
이것은 기존 인공지능이 헷갈려하던 '진짜 악필 중의 악필 데이터'들마저 CNN의 공간 파악 능력을 통해 귀신같이 잡아냈다는 엄청난 성과입니다.

## 6. 실전 코딩 딥다이브: 현업에서는 이것을 어떻게 활용할까?

방금 짠 코드를 기반으로, "만약 내가 내일 회사에 출근해서 강아지와 고양이를 분류하는 CNN AI 앱을 만들어야 한다면?"이라는 가정하에 실무 파이프라인이 어떻게 구성되는지 A to Z를 설명해 드립니다. 

### 실무 활용의 큰 흐름
1.  **데이터 증강(Data Augmentation)**: 실무에서는 언제나 데이터가 부족합니다. 그래서 있는 사진을 살짝 비틀고, 뒤집고, 줌인해서 가짜 데이터를 수만 장 부풀려내는 작업을 코드 앞단에 무조건 추가합니다.
2.  **전이 학습(Transfer Learning)**: 방금 우리는 모델을 밑바닥부터 짰습니다. 하지만 실무에서는 시간이 곧 돈입니다. 구글이 이미 슈퍼컴퓨터로 전 세계 동물 사진을 다 보고 훈련시킨 **특징 추출부(Feature Extractor)**를 공짜로 훔쳐옵니다. 그리고 맨 뒤에 우리가 배운 `Flatten`과 `Dense`만 싹둑 잘라서 내 입맛대로 붙인 뒤 아주 짧게만 학습시킵니다.
3.  **콜백(Callbacks)**: 인공지능이 밤새 학습하다가 가장 똑똑해진 순간의 두뇌 상태(가중치)를 컴퓨터 하드에 저장해야 합니다.

### [실무 응용 예시 코드: 전이학습(VGG16)과 콜백을 적용한 개/고양이 분류기]

```python
import os
import tensorflow as tf
from tensorflow.keras.applications import VGG16
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Flatten, Dense, Dropout
from tensorflow.keras.callbacks import ModelCheckpoint, EarlyStopping

# 1. 빅테크 기업의 위대한 유산(특징 추출기) 빌려오기
# VGG16이라는 구글급 모델을 뼈대로 가져옵니다.
# include_top=False: 맨 뒤에 붙어있던 구글의 1000개짜리 분류기(Dense)는 잘라버리고 가져오겠다는 뜻!
base_model = VGG16(weights='imagenet', include_top=False, input_shape=(224, 224, 3))

# 구글이 고생해서 훈련시켜놓은 앞부분 눈알(특징 추출기)은 시력교정이 끝났으므로 
# 값이 변하지 않게 꽝꽝 얼려줍니다 (Freeze).
base_model.trainable = False

# 2. 내 비즈니스에 맞는 분류기(Classifier) 새로 조립하기
model = Sequential()

# 첫 번째 블록: 얼려놓은 구글의 특급 눈알(Conv+Pool 세트)을 통째로 넣습니다.
model.add(base_model)

# 두 번째 블록: 우리가 배운 분류기 파트입니다.
model.add(Flatten())                     # 공간 정보를 1차원으로 찢고
model.add(Dense(256, activation='relu')) # 내 입맛에 맞게 조합한 뒤
model.add(Dropout(0.5))                  # 과적합을 막고
model.add(Dense(1, activation='sigmoid')) # 개(1) 냐 고양이(0) 냐, 2개 중 하나를 고르는 이진 분류(Sigmoid)를 씁니다.

# 3. 모델 학습 설정
model.compile(
    optimizer='adam',             # 요즘 실무에서 SGD 대신 가장 많이 쓰는 전천후 최적화 알고리즘
    loss='binary_crossentropy',   # 정답이 2개일 때 쓰는 손실 함수 (10개면 categorical_crossentropy)
    metrics=['accuracy']
)

# 4. 스마트한 조력자 콜백(Callbacks) 설정
# EarlyStopping: AI가 학습을 5번(patience) 반복해도 실력이 안 늘면, 밤새 전기세 낭비하지 말고 자동으로 멈추게 합니다.
early_stop = EarlyStopping(monitor='val_loss', patience=5)

# ModelCheckpoint: 학습 중 가장 정답률이 좋았던 최고의 뇌 상태(Best Model)만 파일로 저장해 둡니다.
checkpoint = ModelCheckpoint('best_dog_cat_model.h5', monitor='val_accuracy', save_best_only=True)

# 5. 실전 투입 및 훈련!
# 이제 데이터를 넣고 훈련시킵니다. callbacks에 우리의 조력자들을 명시합니다.

#     callbacks=[early_stop, checkpoint]
```
